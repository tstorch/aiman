# Cornerstone: Software Engineering Best Practices

Dieses Dokument beschreibt verbindliche Arbeitsweisen und dient als Teil des Knowledge Graph.

## Prinzipien

- Klarheit vor Cleverness
- Tests first, kleine Schritte, frühe Integration
- Sicherheit, Datenschutz, Barrierefreiheit berücksichtigen

## Prozesse

- Code Reviews verpflichtend
- ADRs für signifikante Entscheidungen
- Definition of Done pro Ebene (Epic/Feature/Story/Task)

## Qualität

- Linting, Typisierung (wo möglich), automatisierte Tests
- Dokumentation: Frontmatter aktuell halten (`updated`), Querverweise pflegen
